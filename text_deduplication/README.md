# Prompt 文本去重清理工具

一款专为 **零散 Prompt 文本规范化、重复内容批量清理** 设计的 Python 文本处理工具，解决无格式历史文件与规范化新文件之间的内容冗余问题，实现精准匹配、批量删除、一键输出干净文本。

## 项目背景

在日常 Prompt 管理过程中，常常会遇到以下问题：

- 历史积累的 Prompt 文本**散落在多个无格式的 txt 文件**中，无统一结构、难以维护

- 后期重新搭建了**规范化、固定格式的 Prompt 仓库文件**，统一划分对话主题、单行 Prompt、多行 Prompt 模块

- 无格式历史文件中，大量内容已被收录进新的规范化文件，存在大量重复冗余

- 手动逐行查找、比对、删除效率极低，且容易遗漏或误删

本工具的核心目的：**自动在无格式历史文本中，精准识别并删除所有已存在于规范化格式文件中的完整文本片段，实现历史文件轻量化、无冗余清理**。

## 核心功能

1. **规范化文件智能解析**
按照自定义格式规则，自动提取规范化文件中的**对话主题、单行 Prompt、完整多行 Prompt 块**，生成待匹配的清理目标列表。

2. **跨文件精准全文匹配**
对单行文本、连续多行文本块均实现**完整字符级精准匹配**，不忽略空格、换行、符号，仅删除完全一致的重复内容。

3. **批量自动去重清理**
遍历无格式历史文件，自动剔除所有已在规范化文件中存在的内容，保留全新、未收录的有效文本。

4. **结果一键导出**
清理完成后自动生成全新的无冗余文本文件，保留原始格式与有效内容，不破坏原有行文结构。

## 关键处理规则（规范化文件解析逻辑）

工具针对规范化 Prompt 文件的固定格式，设计专属解析规则，精准提取三类核心内容，作为后续清理依据：

### 1\. 注释行跳过规则

- 若行以 `*/` 开头，直接跳过该行，不纳入匹配列表

- 用于过滤规范化文件中的注释、说明文字，避免无效匹配

### 2\. 多行 Prompt 块提取规则

- 检测到以 `/*` 开头的行，自动进入**多行 Prompt 捕获模式**

- 提取 `/*` 之后的所有内容，并持续收集后续所有行

- 直到检测到以 `*/` 结尾的行，结束捕获，将**整块连续文本拼接为一个完整元素**，存入匹配列表

- 适用于规范化文件中**长文本、多轮对话、完整指令块**的精准提取

### 3\. 普通单行内容规则

- 不属于以上两种情况的普通行，去除行尾换行符后，直接作为单行元素存入匹配列表

- 适用于规范化文件中的**对话主题、单行指令、标签、标题**等短文本提取

## 整体工作流程

```Plain Text
1. 读取规范化 Prompt 文件 → 按格式规则解析 → 生成【待删除匹配列表】
2. 读取无格式历史冗余文件 → 加载为完整待清理字符串 S
3. 遍历匹配列表 → 批量删除字符串 S 中所有完全匹配的子串
4. 输出清理后的全新文本 → 保存为独立结果文件
```

## 适用场景

- 历史 Prompt 文本库规范化迁移、冗余清理

- 多文件 Prompt 内容合并、去重、查重

- 零散指令文本统一入库后，旧文件批量瘦身

- 固定格式库与无格式文件之间的重复内容全自动比对删除

## 使用方式

1. 准备两个输入文件：

    - 规范化格式 Prompt 文件（固定结构、已整理好的新文件）

    - 无格式历史冗余文件（待清理的旧文件）

2. 在代码中配置对应文件路径与输出路径

3. 直接运行 Python 脚本，等待执行完成即可得到**无重复、已瘦身的干净文本文件**

## 项目特点

- ✅ **轻量无依赖**：纯 Python 原生代码，无需安装第三方库

- ✅ **精准匹配**：完整字符级匹配，不误删、不漏删

- ✅ **兼容多行块**：支持连续多行文本的整体匹配与删除

- ✅ **格式自适应**：专为 Prompt 管理场景定制解析规则

- ✅ **安全不覆盖**：仅生成新文件，不修改、不覆盖原始输入文件

## 注意事项

- 建议提前备份原始无格式历史文件，避免误操作丢失数据

- 工具采用**完全一致匹配**规则，仅删除百分百重复内容，不会修改或截断原有文本

- 支持 UTF\-8 编码的 txt 文本文件，兼容中英文、特殊符号、换行格式
